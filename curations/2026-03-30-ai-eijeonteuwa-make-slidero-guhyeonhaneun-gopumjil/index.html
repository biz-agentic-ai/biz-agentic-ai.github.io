AI 에이전트와 make-slide로 구현하는 고품질 HTML 슬라이드 자동화

3월 30, 2026 · 2 분 · Junho Lee | 번역:
  • En

당장 내일 오전 미팅인데 손에 쥔 발표 자료가 하나도 없는 상황을 떠올려보자. 내용을 채우는 일만으로도 벅찬데 디자인까지 신경 쓰다 보면 금세 머리가 지끈거린다. Cursor나 Claude Code 같은 도구에 의지해봐도 결과물은 매번 제각각이고, 때로는 엉뚱한 라이브러리를 불러와 빌드 에러를 일으키기도 한다. 프롬프트를 정교하게 다듬는 것만으로는 일관된 품질을 보장하기 어렵다.

Kuneosu가 공개한 make-slide는 바로 이 지점에서 대안을 제시한다. 에이전트가 백지상태에서 디자인을 고민하게 만드는 대신, 사전에 약속된 테마 레퍼런스를 참고하도록 유도한다. npx 명령어로 초기 설정을 마치면 특정 경로에 지침서가 생성되는데, 모델은 이 파일을 읽고 정해진 규칙에 맞춰 HTML 코드를 생성하기 시작한다.

이 방식은 불필요한 비용을 획기적으로 줄여준다. 레이아웃을 말로 구구절절 설명하는 대신 특정 경로의 참조 데이터를 보라고 지시하면 되기 때문이다. 완성된 결과물은 단일 파일 형태로 추출되어 별도의 라이브러리 설치 없이 브라우저만으로 구동된다. 발표 현장에서 단축키를 활용해 노트를 확인하거나 문서를 출력하는 과정도 매끄럽게 이어진다.

작업 방식은 유연하다. 단순한 주제부터 구체적인 텍스트, 혹은 전체 대본까지 어떤 형태의 입력값이든 에이전트가 스스로 분석해 전체적인 구성을 제안한다. 중앙 정렬이나 분할 배치 같은 다양한 레이아웃 중 맥락에 가장 잘 어울리는 스타일을 선택하는 과정이 포함된다. 사용자가 제안된 아웃라인을 검토하고 승인하면 비로소 실제 구현 단계로 진입한다.

전체 워크플로우는 매우 정교하게 설계되어 있다. 무거운 테마 파일을 로컬에 쌓아두지 않고 온라인 저장소에서 필요한 시점에 불러오기에 작업 환경이 쾌적하게 유지된다. 코드 하이라이팅 기능도 기본적으로 내장되어 있어, 엔지니어가 기술적인 내용을 공유할 때 소스 코드를 선명하게 전달하기에 안성맞춤이다.

준비된 스타일의 폭도 넓다. 기술적인 감성이 묻어나는 테마부터 정갈한 비즈니스 형식까지 상황에 맞춰 고를 수 있다. 고화질 이미지를 자동으로 검색해 배치하는 기능은 물론, 오피스 소프트웨어와 호환되는 형식으로 변환하는 규칙도 갖췄다. 명령어를 통해 제공되는 전체 목록을 살펴보면 용도에 맞는 최적의 구성을 쉽게 찾을 수 있다.

조직 내부의 디자인 시스템을 확립하려는 유통사나 금융사라면 전용 테마를 추가해 표준화된 템플릿을 운영할 수 있다. 참조용 파일과 가이드라인만 갖춰두면 에이전트가 이를 엄격히 준수한다. 이는 결과적으로 디자인팀과 개발팀 사이의 시각적 차이를 좁히는 유용한 도구가 된다.

이는 단순한 저작 도구를 넘어선다. 자동화 시스템에 특정 분야의 지식이나 디자인 원칙을 어떻게 학습시킬지에 대한 방법론을 보여준다. 설정 파일에 해당 기능을 연결해두면 명령어 한 줄로 복잡한 작업이 마무리된다. 일일이 내용을 복사하고 붙여넣던 단순 반복 작업에서 벗어나는 쾌감을 선사한다.

데이터 중심의 구조 덕분에 세부적인 수정도 용이하다. 별도의 라이브러리 없이도 막대그래프나 지표 카드를 시각화하는 기능을 활용할 수 있다. 파일 자체가 가벼워 온라인 저장소에 업로드한 뒤 즉시 웹상에서 공유하기에도 무리가 없다.

이어지는 글에서는 데이터 간의 관계를 시각적으로 풀어내는 자동화 기법에 대해 논의해볼 예정이다.

이 글에서 건진 것들

  • AI에게 디자인 결정을 맡기지 않고 사전 정의된 레퍼런스를 참조하게 하여 토큰 효율과 품질 일관성을 확보했다.
  • 단일 HTML 파일 출력 방식은 의존성 문제를 해결하며 브라우저만 있다면 어디서든 발표가 가능한 범용성을 제공한다.
  • .claude/skills/와 같은 에이전트 전용 지침서 시스템을 활용해 복잡한 워크플로우를 명령 하나로 자동화하는 패턴이 유효하다.
